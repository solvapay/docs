---
title: "Create a Product"
description: "Use this flow if you are a developer building with the SDK or shipping product access without Hosted MCP Pay."
---

This product flow is ideal for:

* SDK-backed MCP server products
* Chat agent access products
* API access and app feature monetization

By default, new products are available for **SDK Integration** and active in **SolvaPay MCP Distribution**.

## Prerequisites[​](#prerequisites "Direct link to Prerequisites")

* SolvaPay provider account
* At least one plan already created (or created during this flow)

## Step 1: Open Products[​](#step-1-open-products "Direct link to Step 1: Open Products")

Go to **Products** in the provider dashboard and start product creation.

![Products dashboard with create product entry point](/images/getting-started/assets/images/regular-step1-products-list-a098c915342729b2e74e91c1f7f99fbf.png)

## Step 2: Fill Product Details[​](#step-2-fill-product-details "Direct link to Step 2: Fill Product Details")

Enter your product details:

* Name
* Description
* Optional tags and image

![Product details step in create product flow](/images/getting-started/assets/images/regular-step2-product-details-a86f8bc280fd3339277f876d34e385f8.png)

## Step 3: Select Plans and Create Product[​](#step-3-select-plans-and-create-product "Direct link to Step 3: Select Plans and Create Product")

In the Plans step:

1. Select one or more existing plans, or create a new plan inline
2. Confirm selected plans
3. Click **Create product**

Once created, your product is active and ready for SDK-led integration workflows.

## Result: Product Is Ready by Default[​](#result-product-is-ready-by-default "Direct link to Result: Product Is Ready by Default")

On the product details page, you can confirm:

* Product is active
* Plans are attached
* Integration options are available

![Product details page after successful product creation](/images/getting-started/assets/images/regular-step3-product-created-2233d9ee12c123fd16510a4198362015.png)

## What to Do Next[​](#what-to-do-next "Direct link to What to Do Next")

* Integrate checkout/access control with the SDK
* Share product access according to your selected plans
* Validate purchases and usage in sandbox before going live
